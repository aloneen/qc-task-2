### Дополнительно: сравнение с enum

    public enum ConfigEnum {
        INSTANCE;
    
        private final String appName = "MyApp";
    
        public String getAppName() {
            return appName;
        }
    }

| Характеристика              | Singleton на классе (Holder)                 | Singleton через enum                |
|-----------------------------|----------------------------------------------|-------------------------------------|
| Объём кода                  | Больше вспомогательной логики                | Минимальный синтаксис               |
| Инициализация               | Происходит при первом обращении              | Происходит при загрузке enum        |
| Работа в многопоточности    | Безопасен за счёт механизма загрузки классов | Безопасен по спецификации JVM       |
| Поведение при сериализации  | Требует переопределения readResolve          | Корректен без доп. кода             |
| Защита от reflection        | Реализуется вручную                          | Гарантируется JVM                   |
| Возможность расширения      | Допускает дополнительную логику              | Ограничена конструкцией enum        |
| Применимость для конфигов   | Хорошо подходит                              | Подходит не всегда                  |
